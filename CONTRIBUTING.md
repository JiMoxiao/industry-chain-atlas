# Contributing

感谢你关注 `Semiconductor Industry Chain Atlas`。

这个项目欢迎以下类型的贡献：

- 修复 bug
- 优化图谱交互与页面体验
- 提升数据生成、缓存或加载性能
- 补充新的产业链 YAML 数据
- 改进文档、启动脚本和跨平台支持

## 开始之前

建议先完成以下准备：

1. Fork 仓库并创建自己的分支
2. 按照 [README.md](./README.md) 完成项目启动
3. 确认修改范围和影响面

推荐分支命名：

- `feat/<short-name>`
- `fix/<short-name>`
- `docs/<short-name>`
- `refactor/<short-name>`

## 提交 Issue

如果你要反馈问题，建议尽量包含以下信息：

- 操作系统和运行环境
- Python、Node.js、pnpm 版本
- 复现步骤
- 实际结果与预期结果
- 控制台报错、接口报错或截图

如果问题与数据有关，也请说明涉及的产业链 slug 或 YAML 文件。

## 提交 Pull Request

建议遵循下面的流程：

1. 先创建 Issue 或在已有 Issue 下认领任务
2. 在独立分支中完成改动
3. 自测通过后再提交 PR
4. 在 PR 描述中写清：
   - 改了什么
   - 为什么改
   - 如何验证
   - 是否影响数据结构、接口或页面行为

## 本地开发建议

### 启动项目

Windows 用户优先使用：

```bat
start.bat
```

双击 `start.bat` 即可；如需调试底层脚本，也可继续使用 `.\start.ps1`。其余情况下按 README 中的手动方式分别启动前后端。

### 常用检查

前端构建：

```bash
cd web
pnpm build
```

前端类型检查：

```bash
cd web
pnpm check
```

重新生成数据：

```bash
python generate_data.py --all
```

刷新研究数据：

```bat
refresh-data.bat
```

## 数据相关约定

- `data/*.yaml` 是研究链路与结构信息的源数据
- 生成脚本会把结果写入 `web/src/data/`
- 后端优先读取已生成 JSON，并在进程内做缓存
- 如果你修改了 YAML、趋势、审计或热度相关逻辑，请同步完成数据重生成和页面验证

## 代码风格建议

- 保持改动聚焦，不要在同一个 PR 中混入大量无关修改
- 优先修复根因，而不是只做表层绕过
- 文档、命令和实际行为要保持一致
- 若涉及 UI 改动，建议附上截图或录屏
- 若涉及接口或数据结构变更，请在 README 或相关文档中补充说明

## 提交前检查清单

提交 PR 前，建议至少确认：

- 项目可以正常启动
- `pnpm build` 通过
- 关键页面可正常访问
- 受影响的数据链路已验证
- 文档已同步更新

## 不建议提交的内容

请不要提交以下本地产物：

- `node_modules/`
- `dist/`
- `.dbg/`
- Python 缓存目录
- 本地浏览器缓存或调试生成文件

## 行为预期

欢迎理性、尊重、可复现的问题反馈和代码讨论。

如果你准备做较大范围的重构，建议先开 Issue 讨论设计方向，再开始实现。
